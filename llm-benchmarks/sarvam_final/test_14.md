**Bug(s) in the original code**

```python
def group_by_first_letter(words, groups={}):
    for w in words:

```

1. **Mutable default argument** – `groups={}` is created once when the function is defined. The same dictionary is reused on every call.  
   *Consequence*: each call to `group_by_first_letter` adds its new words to the *same* dictionary, so the result “accumulates” across calls.

2. **Side‑effects** – The function mutates the `groups` argument (or the default) and returns it. Callers that expect a fresh grouping for each call are surprised by the shared state.

3. **Potential `KeyError`** – If a word is an empty string (`''`) the expression `w[0]` raises `IndexError`. (Not the main reported problem, but a robustness issue.)

---

**Root cause**

The mutable default argument is the classic Python pitfall. Because the dictionary lives at function‑definition time, it is shared across all invocations. The `setdefault` call mutates that shared dict, so subsequent calls see the data from previous calls.

---

**Corrected version**

The fix is to avoid using a mutable default. Instead, create a fresh dictionary inside the function (or require the caller to pass one). A clean, side‑effect‑free implementation:

```python
def group_by_first_letter(words, groups=None):
    """
    Return a dict mapping each first letter to a list of words that start with that letter.

    Parameters
    ----------
    words : iterable of str
        The words to be grouped.
    groups : dict, optional
        An existing dict to populate. If omitted, a new dict is created.

    Returns
    -------
    dict
        Mapping from first letter to list of words.
    """
    if groups is None:
        groups = {}                     # fresh dict for this call

    for w in words:
        # Guard against empty strings – skip or handle as you prefer.
        if not w:

        groups.setdefault(w[0], []).append(w)

```

* `groups=None` is a sentinel that tells the function “no dict supplied”.  
* Inside the function we create a *new* dictionary (`groups = {}`) each time, so the result is isolated to that call.
